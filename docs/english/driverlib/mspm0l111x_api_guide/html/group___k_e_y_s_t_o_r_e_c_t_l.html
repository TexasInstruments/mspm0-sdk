Detailed Description

Overview

The Keystore Controller (KEYSTORECTL) DriverLib allows full configuration of the MSPM0 KEYSTORECTL Module.

enum DL_KEYSTORECTL_STATUS
Enumerator
DL_KEYSTORECTL_STATUS_VALID 

Valid status bit. Configuration is valid or operation completed.

DL_KEYSTORECTL_STATUS_NO_CONFIG 

No Configuration Status bit, no number of 256 keys has been identified. The user must call DL_KEYSTORECTL_setNumberOf256Keys.

DL_KEYSTORECTL_STATUS_INVALID_NUM_KEYS 

Invalid number set during DL_KEYSTORECTL_setNumberOf256Keys

DL_KEYSTORECTL_STATUS_BUSY_RX 

Busy RX status bit. The engine is busy receiving a key deposit.

DL_KEYSTORECTL_STATUS_BUSY_TX 

Busy TX status bit. The engine is busy transmitting a key to a crypto engine

DL_KEYSTORECTL_STATUS_INVALID_SLOT_WR 

Invalid key write configuration. This means one or more of the following: the selected slot exceeds available slots, the selected slot already contains a valid programmed key, the selected slot has a mismatched key size compared to the KEYSTORECTL configuration.

DL_KEYSTORECTL_STATUS_INVALID_SLOT_RD 

Invalid key read configuration (transmitting a key). This means one or more of the following: the selected slot exceeds available slots, the selected slot does not contain a valid programmed key, the selected slot has a mismatched key size

DL_KEYSTORECTL_STATUS_WRITE_FAILED 

Key slots are not valid after a write. Write failed during the DL_KEYSTORECTL_writeKey (driver library specific).

§ DL_KEYSTORECTL_setNumberOf256Keys()

__STATIC_INLINE void DL_KEYSTORECTL_setNumberOf256Keys ( KEYSTORECTL_Regs *  keystorectl,
DL_KEYSTORECTL_NUM_256_KEYS  numKeys 
)

Sets the number of combined 256 key slots.

For a given number of 128-bit key slots n, there can be up to n/2 slots combined to work as one 256-bit key slot. These larger slots are added to earlier slots first. Thus, a 4-slot device with one 256-key slot will have slots 0 and 1 combined, and slot 2 and 3 will remain 128-bit key slots. Refer to the device datasheet for the specific number of key slots.

Parameters
[in]keystorectlPointer to the register overlay for the peripheral
[in]numKeysNumber of 256-bit keys, one of DL_KEYSTORECTL_NUM_256_KEYS. Must be less than the total key slots on the device divided by two
Note
this function can only be written in a secure operating state before the INITDONE signal is asserted.

__STATIC_INLINE void DL_KEYSTORECTL_setDestinationKeySlot ( KEYSTORECTL_Regs *  keystorectl,

Sets the intended destination key slot for the next write.

Sets the destination key slot for a data write to that region. In the case of 128-bit key slot, only one slot should be selected. In the case of a 256- bit key slot, both registers should be selected and the key size should also reflect this change. If 256- it keys are desired, they should be configured using DL_KEYSTORECTL_NUM_256_KEYS. Refer to the device datasheet for the specific number of key slots.

Parameters
[in]keystorectlPointer to the register overlay for the peripheral
[in]keySlotThe intended key slot. Can be 128-bit or a 256-bit. One of DL_KEYSTORECTL_KEY_SLOT
Note
this function can only be written in a secure operating state before the INITDONE signal is asserted.

§ DL_KEYSTORECTL_getNumSlots()

__STATIC_INLINE DL_KEYSTORECTL_NUM_SLOTS DL_KEYSTORECTL_getNumSlots ( const KEYSTORECTL_Regs *  keystorectl)

Gets the total number of 128-bit key slots in HW on the device.

This is the number of key slots that the KEYSTORECTL module has in HW on the device. It does not reflect the configuration or the number of 256-bit keys, only the total number of slots. This also governs the valid configurations, and the number of 256-bit keys must be less than or equal to half the value.

Parameters
[in]keystorectlPointer to the register overlay for the peripheral
Returns
The number of 128-bit slots on the device. One of DL_KEYSTORECTL_NUM_SLOTS

§ DL_KEYSTORECTL_writeKey()

DL_KEYSTORECTL_STATUS DL_KEYSTORECTL_writeKey ( KEYSTORECTL_Regs *  keystorectl,
DL_KEYSTORECTL_KeyWrConfig *  keyWrConfig 
)

Writes a key into the KEYSTORE.

The sequence this function will follow:

  1. Confirm valid status on the KEYSTORECTL
  2. Configure the key write by writing size and slot to the key that is to be written. Passed in via DL_KEYSTORECTL_KeyWrConfig
  3. Confirm a valid write configuration
  4. Write key to the input buffer
  5. Loop until status is back to valid
  6. Confirm key slots are successfully written
Parameters
[in]keystorectlPointer to the register overlay for the peripheral
[in]keyWrConfigPointer to the write key configuration struct DL_KEYSTORECTL_KeyWrConfig
Returns
Status of the operation of type DL_KEYSTORECTL_STATUS. Type will be DL_KEYSTORECTL_STATUS_VALID if successful, or return the error status.
Note
The function DL_KEYSTORECTL_setNumberOf256Keys must be run before this can be executed.
It is not possible to overwrite a previously configured key. A full BOOTRST must be executed before the KEYSTORECTL is erased and can be re-initialized and written.
this function can only be written in a secure operating state before the INITDONE signal is asserted.

§ DL_KEYSTORECTL_transferKey()

DL_KEYSTORECTL_STATUS DL_KEYSTORECTL_transferKey ( KEYSTORECTL_Regs *  keystorectl,
const DL_KEYSTORECTL_Config *  config 
)

Transfers key loaded in KEYSTORE into intended crypto.

Combination of previous functions in order to give the entire operation. Expected runtime sequence:

  1. Confirm valid status on the KEYSTORECTL
  2. Configure the read transfer by writing size, slot, and recipient This is passed in via the struct DL_KEYSTORECTL_Config
  3. Confirm a valid status and transfer configuration of the KEYSTORECTL
  4. Loop until status is back to valid
Parameters
[in]keystorectlPointer to the register overlay for the peripheral
[in]configPointer to a filled out config struct of type DL_KEYSTORECTL_Config
Returns
Status of the operation of type DL_KEYSTORECTL_STATUS. Type will be DL_KEYSTORECTL_STATUS_VALID if successful, or return the error status.
