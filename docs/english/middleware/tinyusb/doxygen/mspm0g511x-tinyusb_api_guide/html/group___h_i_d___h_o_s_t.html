Set protocol mode for a HID device.

This function configures the protocol mode for the specified HID interface. It is only supported by Boot interfaces (where tuh_hid_interface_protocol() != HID_ITF_PROTOCOL_NONE). The function sends a SET_PROTOCOL control request to the device.

Parameters
[in]dev_addrDevice address
[in]idxInstance number relative to the device
[in]protocolProtocol mode to set:
  • HID_PROTOCOL_BOOT (0): Boot Protocol
  • HID_PROTOCOL_REPORT (1): Report Protocol
Returns
true if the request was successfully sent false if the interface doesn't support protocol switching or request failed

§ tuh_hid_send_report()

bool tuh_hid_send_report ( uint8_t  dev_addr,
uint8_t  idx,
uint8_t  report_id,
const void *  report,
uint16_t  len 
)

Send report to HID device using interrupt endpoint.

This function sends a report to the specified HID interface using its interrupt OUT endpoint. When the report is successfully delivered, the tuh_hid_report_sent_cb() callback will be invoked.

Parameters
[in]dev_addrDevice address
[in]idxInstance number relative to the device
[in]report_idReport ID (0 if report IDs are not used)
[in]reportBuffer containing the report data to send
[in]lenLength of the report data in bytes
Returns
true if the send transfer was successfully queued false if the interface doesn't have an OUT endpoint or the transfer failed
Note
If report_id > 0, it will be sent as the first byte followed by the report contents. If report_id = 0, only the report content is sent.

§ hidh_set_config()

bool hidh_set_config ( uint8_t  dev_addr,
uint8_t  itf_num 
)

Configure a HID interface after device configuration.

This internal function is called after the device's configuration has been set. It initiates the HID-specific configuration sequence by creating a control transfer that starts the multi-step process of:

  1. SET_IDLE (to disable continuous reports)
  2. SET_PROTOCOL (for boot devices)
  3. GET_REPORT_DESCRIPTOR
Parameters
[in]dev_addrDevice address
[in]itf_numInterface number
Returns
true if the configuration process was successfully initiated
false if the interface is not available or configuration failed
